Web-based mapping done right.

Meridian wraps code into reusable and decoupled components and/or extensions that can easily communicate with each other.

Components

A Meridian component is reusable, independent code that works well with other components or on its own. Every unit of a page can be represented as a component. Using the component source, you can add the components on your page on demand with data attributes.

See More >

Messaging

Meridian Messaging allows components to communicate between each other. Publish, subscribe, and/or unsubscribe to sandboxed events emitted by other components.

See More >

Sandboxing

A Meridian sandbox is the layer between each component, libraries, and the Meridian Core. It protects and controls component scope, provides flexibility to switch out libraries with greater ease and allows global access to common functionality.

See More >

Extensions

Meridian Extensions help you extend the core capabilities of your app before any of your components load. It'll make them available via the component's sandbox so you don't have to worry about importing and setting this up yourself.

See More >
